# crypto-news-automation 发布检查清单

每次修改代码或发布前，必须执行以下检查：

---

## 🚨 安全检查项（必须通过）

每次发布到 GitHub 前，必须确认：

- [ ] **不提交 state/** - `state/` 目录已加入 `.gitignore`，不得提交
- [ ] **不提交真实 config/config.json** - 只能提交 `config.example.json`
- [ ] **不打印真实 API Key** - `scripts/*.js` 中打印 key 前后缀时必须用 `if (process.env.DEBUG === 'true')` 包裹
- [ ] **文档中只允许占位符** - README/SKILL.md/NOTES.md 中只能出现 `YOUR_API_KEY`、`your_api_key` 等占位符，禁止真实凭据

---

## ⚠️ 发布质量检查（2026-03-10新增）

每次发布前，必须确认内容符合最低可发布标准：

### 最低可发布标准
- [ ] 有标题（含日期）
- [ ] 有【资讯速递】且至少 3 条
- [ ] 有【技术面分析】且至少 3 个币
- [ ] 有完整结构（不是简单涨跌幅拼接）
- [ ] 不像占位文案/测试样例

### 遇到风控时的处理
- [ ] ❌ 不得自动降级到"简化版"/"测试版"内容
- [ ] ✅ 生成温和改写版，发给用户审核
- [ ] ✅ 用户确认后才允许发布

### 禁止的降级内容
以下情况必须拒绝发布：
- [ ] 只剩几个币种涨跌幅
- [ ] 没有资讯内容
- [ ] 没有完整结构
- [ ] 像占位文案/垃圾帖/测试帖
- [ ] 只有一句"关注哪个币？"

---

## 📝 代码检查项

每次修改 generate-article.js 后，必须执行以下检查：

## 1. 运行测试
```bash
cd ~/.openclaw/workspace/.agents/skills/crypto-news-automation
node scripts/generate-article.js
```

## 2. 对照SKILL.md模板检查输出

### 资讯速递格式
- [ ] 有标题 `📰 加密货币日报 · YYYY-MM-DD`
- [ ] 有分隔线 `---`
- [ ] 有 `【资讯速递】` 标题
- [ ] 5条资讯，每条单独编号
- [ ] **资讯是中文**（有翻译或用AI翻译）
- [ ] **5条都与加密货币直接相关**（禁止 Zoom / 办公软件 / 消费电子 / 泛科技混入）
- [ ] **无占位标题**（如 “Here’s what happened in crypto today” / “以下是今天加密货币发生的事情”）

### 技术面分析格式
- [ ] 有 `【技术面分析 - 24h热门波动币】` 标题
- [ ] 5个币种，每个格式如下（4行）：
```
1. $ETH (Ethereum)
   当前价格: $2,000 | 24h涨跌: +2.5%
   24h成交量: $20B | 市值排名: #2
   走势判断: 📈 看涨 | 小幅震荡
```

### 结尾
- [ ] 有分隔线 `---`
- [ ] 有 `回复「确认发送」→ 发布到币安`

## 3. 推送测试
用 feishu-send.sh 测试发送到群
